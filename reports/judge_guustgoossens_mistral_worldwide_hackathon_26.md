## Summary

Summa turns any codebase into a living 3D knowledge graph that can be explored visually and interrogated via voice. It parses source code using web-tree-sitter (WASM) into an AST, enriches it with git history analysis, loads everything into KuzuDB (an embedded graph database running in WebAssembly in-browser), and connects a voice interview system powered by ElevenLabs Conversational AI and Mistral (DevStral Small 2 for real-time voice, DevStral 2 for deep multi-step graph reasoning). The system also ships an MCP server for integration with Claude Code and Cursor, and includes integration with `voxtral.c` for local on-device STT via Apple Silicon Metal.

## Strengths

- In-browser KuzuDB (WASM) with multi-language tree-sitter parsing is a technically novel combination — giving the AI agent one generic tool (`query_graph(cypher)`) that can answer any structural code question is architecturally elegant
- The multi-step reasoning agent (`server/reasoner/agent-loop.ts`) with L0→L1→L2 progressive disclosure, Cypher query composition by the 123B DevStral 2, loop detection, and MAX_STEPS=8 with forced conclusion shows production-quality AI agent design
- The two-speed Mistral architecture (DevStral Small 2 at ~200 tokens/s for voice, DevStral 2 at ~76 tokens/s for deep analysis) is a principled latency/capability tradeoff with the right model for each task
- Pre-computed interview briefing (6 parallel KuzuDB Cypher queries → DevStral Small 2 JSON briefing → stored on proxy as session context) solves the ElevenLabs tool round-trip reliability problem gracefully
- Graph schema (`File`, `Function`, `Class`, `Person`, `Discussion` nodes; `CONTAINS`, `CALLS`, `IMPORTS`, `INHERITS`, `CONTRIBUTED`, `UNDERSTANDS`, `HAS_PARTICIPANT`, `ABOUT` relationships) captures code architecture, contributor ownership, and knowledge topology in a unified model
- Integration with `voxtral.c` for local Apple Silicon Metal STT, AWS Bedrock dual-mode inference, and MCP server for Claude Code / Cursor — excellent sponsor integration breadth
- Live deployment at https://mistral-worldwide-hackathon-26.vercel.app/ with pre-computed graph data for the `mistral-vibe` and `mistral-python-client` repos

## Areas for Improvement

- The ElevenLabs tool round-trip issue (intermittent failures when the voice agent tries to compose Cypher queries in real time) forced a pivot to pre-computed briefings — while well-handled, this means the voice agent cannot dynamically query the graph during conversation, limiting the live intelligence
- Only the Interview Agent and Quiz System reached full implementation; the Graph Reasoner and Background Enricher exist as server-side components but are not accessible via the voice interface
- The `voxtral.c` integration requires running a separate native process on Apple Silicon — complex setup that won't work on most deployment targets
- The README promises a Knowledge overlay showing green/red/yellow nodes based on quiz performance, but verifying this works end-to-end requires running the full pipeline locally
- Solo developer project with ambitious scope — the 34-hour timeline shows in some areas (the `UNDERSTANDS` relationship is populated by the quiz system but the People overlay for knowledge silos may not be fully rendered)

## Detailed Reasoning

Summa is one of the most architecturally sophisticated projects in this batch. The central insight — that a codebase is fundamentally a graph (functions call functions, files import files, people contribute to overlapping systems) and that trees capture syntax but graphs capture relationships — is both intellectually compelling and practically useful. The choice of KuzuDB as an embedded WASM graph database is excellent: it runs entirely in the browser with no server required for graph queries, yet exposes a full Cypher query interface that AI agents can compose dynamically.

The `server/enricher/index.ts` pipeline demonstrates a mature understanding of AI-assisted metadata enrichment: `enrichNodeSummaries()` generates 50-token L0 summaries for each `Function`, `File`, and `Class` node using DevStral Small 2 in batch mode, while `enrichRelationshipL1()` generates relationship summaries for edges (contributor history, call patterns, import relationships). These L0/L1 summaries are the foundation of the progressive disclosure design: the voice agent's pre-interview briefing queries L0 summaries to build a compact overview of the codebase structure, which is stored as the session system message. This avoids the ElevenLabs round-trip latency problem while still giving the interview agent codebase-aware context.

The `server/reasoner/agent-loop.ts` is architecturally elegant. It implements a MAX_STEPS=8 reasoning loop where DevStral 2 (123B) composes Cypher queries at each step, specifying a `dimension` (structural/contribution/knowledge) and `level` (L0/L1/L2/L3). The `LoopDetector` tracks repeated queries (using string similarity) and repeated empty results, injecting loop-breaking instructions when patterns are detected. When the agent signals `done: true`, it provides a final answer and optional `nodeIds` for graph highlighting. This design gives the agent genuine multi-step analytical capability for complex questions like "who should fix this auth bug?" without the hallucination risk of a single-call answer.

The graph schema is particularly thoughtful. `Person` nodes are described as "invisible infrastructure" — they exist for powerful relationship queries (who contributed to which files, who understands which functions) but don't render in the default 3D view, surfacing only through overlay modes. The `UNDERSTANDS` relationship is populated by the quiz system (question generation → user answer → score update → graph write), creating a real-time knowledge map that evolves as users interact with the codebase. The four overlay modes (Structure, Contributors, Knowledge, People) each query the same KuzuDB graph with different relationship traversals to derive visualization-ready `{nodes, links}` data.

The solo development context (34 hours, one developer) explains the scope gaps: the full 4-agent architecture (Interview Agent, Quiz System, Graph Reasoner, Background Enricher) was designed but only the first two reached full frontend integration. The Graph Reasoner and Background Enricher work as standalone server-side scripts but aren't yet connected to the voice interface. The documented pivot from real-time Cypher query generation during voice sessions to pre-computed briefings (in `docs/architecture/ARCHITECTURE_CHANGES.md`) shows honest engineering judgment — reliability over magic.
